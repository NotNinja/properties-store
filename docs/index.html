PropertiesStore

src/properties-store.js

A PropertiesStore represents a persistent set of properties which can be loaded from and/or saved to a stream. Each property key and their corresponding value is a string which can be iterated over as PropertiesStore itself is iterable.

It is designed to be compatible with Java's .properties file format, however, it also has the ability to maintain non-properties lines that are loaded from an input stream, which can be useful when wanting to make non-intrusive changes to the original source.

The preserve option can be enabled so that all lines read by PropertiesStore#load are maintained and also written by PropertiesStore#store, but it's worth noting that doing so can slow certain operations while PropertiesStore maintains synchronization with the property lines amongst the other preserved lines.

Optionally, another store can be specified whose properties (and lines, if preserved) will be used as the base for the new PropertiesStore instance.

new PropertiesStore(store: PropertiesStore?, options: Object?)

Extends events.EventEmitter

Parameters
store (PropertiesStore?) a PropertiesStore whose properties and lines, where applicable, are to be used initially
options (Object?) the options to be used
Name Description
options.preserve boolean (default false) true to preserve all lines read by PropertiesStore#load , even those that do not represent properties; otherwise false
Example
const PropertiesStore = require('properties-store');

const properties = new PropertiesStore();
properties.set('foo', 'bar');
properties.set('fu', 'baz');

properties.get('foo');
//=> "bar"
properties.get('fu');
//=> "baz"

const copy = new PropertiesStore(store);
Array.from(copy);
//=> [["foo", "bar"], ["fu", "baz"]]
Static Members
▸ load(input, options?)
src/properties-store.js

Reads the property information from the input stream provided and loads it into a new PropertiesStore.

If property lines are found with the same key in input, the value of the latest property line with that key will be set as the value.

The preserve option can be enabled so that all lines read by this method are maintained and also written by PropertiesStore#store, but it's worth noting that doing so can slow certain operations while PropertiesStore maintains synchronization with the property lines amongst the other preserved lines.

By default, any Unicode escapes ("\uxxxx" notation) read from input will be converted to their corresponding Unicode characters. This behaviour can be prevented by disabling the unescape option.

load(input: stream.Readable, options: Object?): Promise<PropertiesStore, Error>
Parameters
input (stream.Readable) the input stream from which the properties are to be read
options (Object?) the options to be used
Name Description
options.encoding string (default "latin1") the character encoding to be used to read the input
options.preserve boolean (default false) true to preserve all lines read, even those that do not represent properties; otherwise false
options.unescape boolean (default true) true to convert all Unicode escapes ("\uxxxx" notation) to their corresponding Unicode characters; otherwise false
Returns
Promise<PropertiesStore, Error>: A Promise that is resolved with the new PropertiesStore once input has be read.
Example
const properties = await PropertiesStore.load(fs.createReadStream('path/to/my.properties'), {
  encoding: 'utf8',
  unescape: false
});
Array.from(properties);
//=> [["foo", "bar"], ["fu", "baz"]]
Array.from(properties.lines());
//> ["foo = bar", "fu = baz"]

const completeProperties = await PropertiesStore.load(fs.createReadStream('path/to/my.properties'), {
  encoding: 'utf8',
  preserve: true,
  unescape: false
});
Array.from(completeProperties);
//=> [["foo", "bar"], ["fu", "baz"]]
Array.from(completeProperties.lines());
//> ["# My Properties", "", "foo = bar", "fu = baz"]

▸ load(input, options?)
src/properties-store.js

Reads the property information from the input stream provided and loads it into this PropertiesStore.

If property lines are found with the same key in either this PropertiesStore or input, the value of the latest property line with that key will be set as the value.

If this PropertiesStore is preserving all lines, they will all be loaded from input and maintained within this PropertiesStore, regardless of whether or not they represent a property. Each line loaded will be added after any previously loaded lines, where applicable.

By default, any Unicode escapes ("\uxxxx" notation) read from input will be converted to their corresponding Unicode characters. This behaviour can be prevented by disabling the unescape option.

load(input: stream.Readable, options: Object?): Promise<void, Error>
Parameters
input (stream.Readable) the input stream from which the properties are to be read
options (Object?) the options to be used
Name Description
options.encoding string (default "latin1") the character encoding to be used to read the input
options.unescape boolean (default true) true to convert all Unicode escapes ("\uxxxx" notation) to their corresponding Unicode characters; otherwise false
Returns
Promise<void, Error>: A Promise that is resolved once input has be read into this PropertiesStore .
Example
const properties = new PropertiesStore()

await properties.load(fs.createReadStream('path/to/my.properties'), {
  encoding: 'utf8',
  unescape: false
});
Array.from(properties);
//=> [["foo", "bar"], ["fu", "baz"]]
Array.from(properties.lines());
//> ["foo = bar", "fu = baz"]

  encoding: 'utf8',
  unescape: false
});

//=> [["foo", "bar"], ["fu", "baz"]]
Array.from(completeProperties.lines());
//> ["# My Properties", "", "foo = bar", "fu = baz"]
▸ set(key, value)
src/properties-store.js

Sets the value of the property in this PropertiesStore with the specified key to value.

key is case sensitive.

Nothing happens if key is null. If value is null, PropertiesStore#delete will be called to removed the property.

If this PropertiesStore is preserving all lines, the last line representing the property with key will be updated with value, if one exists, otherwise a new line will be added for the property. If value is null, each line representing the property with key will be removed.

set(key: string?, value: any): PropertiesStore
Parameters
key (string?) the key of the property whose value is to be set (may be null )
value (any) the new value for the property (will be cast to a string without leading whitespace - may be null to remove property instead)
Returns
PropertiesStore: A reference to this PropertiesStore .
Example
const properties = new PropertiesStore();

properties
  .set('foo', 'bar')
  .set('fu', 'baz');
Array.from(properties);
//=> [["foo", "bar"], ["fu", "baz"]]

properties.set('FOO', 'BAR');
Array.from(properties);
//=> [["foo", "bar"], ["fu", "baz"], ["FOO", "BAR"]]

properties.set('FOO', null);
Array.from(properties);
//=> [["foo", "bar"], ["fu", "baz"]]

properties.set('foo', 'BAR');
Array.from(properties);
//=> [["foo", "BAR"], ["fu", "baz"]]

▸ store(output, options?)
src/properties-store.js

Writes the property information within this PropertiesStore to the output stream provided.

If this PropertiesStore is preserving all lines, they will all be written directly to output. Otherwise, only lines for properties within this PropertiesStore will be written to output.

By default, any characters that are not part of the ASCII character set will be converted to Unicode escapes ("\uxxxx" notation) before being written to output. This behaviour can be prevented by disabling the escape option.

store(output: stream.Writable, options: Object?): Promise<void, Error>
Parameters
output (stream.Writable) the output stream to which the properties are to be written
options (Object?) the options to be used
Name Description
options.encoding string (default "latin1") the character encoding to be used to write the output
options.escape boolean (default true) true to convert all non-ASCII characters to Unicode escapes ("\uxxxx" notation); otherwise false
Returns
Promise<void, Error>: A Promise that is resolved once this PropertiesStore has been written to output .
Example
const properties = new PropertiesStore();
await properties.load(fs.createReadStream('path/to/my.properties'));
properties.set('foo', 'BAR');
properties.set('fu', null);

await properties.store(fs.createWriteStream('path/to/my.properties'));
fs.readFileSync('path/to/my.properties', 'latin1');
//=> "foo = BAR"

const completeProperties = new PropertiesStore({ preserve: true });
await completeProperties.load(fs.createReadStream('path/to/my.properties'));
completeProperties.set('foo', 'BAR');
completeProperties.set('fu', null);

await completeProperties.store(fs.createWriteStream('path/to/my.properties'));
fs.readFileSync('path/to/my.properties', 'latin1');
//=> "# My Properties

foo = BAR"
